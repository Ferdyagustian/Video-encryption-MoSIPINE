1. ARSITEKTUR SISTEM & DAFTAR FILE
================================================================================

Sistem ini adalah aplikasi desktop kriptografi video dan audio berbasis
peta chaos (chaotic map) dengan antarmuka GUI Tkinter. Sistem menggunakan
operasi XOR antara data multimedia dan keystream pseudo-random yang
dibangkitkan oleh peta chaos Modulo Sine-PWLCM.

  gui_video_main.py  — Entry point GUI utama (main window)

  generator_nist.py  — Generator bitstream 1 Megabit untuk uji NIST STS
  bifurkasi.py       — Skrip visualisasi diagram bifurkasi
  lyapunov.py        — Skrip visualisasi eksponen Lyapunov
  fix_avi.py         — Skrip utilitas untuk migrasi format dari .avi ke .mp4

Direktori & File Pendukung:

2. sipine.py — INTI ALGORITMA CHAOTIC MAP
================================================================================

File ini berisi implementasi inti pembangkit keystream chaos yang
dikompilasi menggunakan Numba JIT untuk performa tinggi (50-100x lebih
cepat dari pure Python). TIDAK BOLEH DIMODIFIKASI.

2.1 PETA CHAOS YANG TERSEDIA (3 MODE):

  Mode 0 — MO-SiPINE (Modulo Sine-PWLCM, default):
    Komposisi dari Piecewise Linear Chaotic Map (PWLCM) dan fungsi Sine.
    Rumus: x_{n+1} = (beta * sin(pi * F_PWLCM(x_n)) * 987676766.3232) mod 1.0
    dimana F_PWLCM adalah:
      Jika x >= 0.5 → evaluasi (1-x)
      Jika 0 <= x < p → f(x) = x/p
      Jika p <= x < 0.5 → f(x) = (x-p)/(0.5-p)

  Mode 1 — Sine Only:
    x_{n+1} = (beta * sin(pi * x_n) * 987676766.3232) mod 1.0

  Mode 2 — PWLCM Only:
    x_{n+1} = (F_PWLCM(x_n) * 987676766.3232) mod 1.0

2.2 MEKANISME PEMBANGKITAN KEYSTREAM:

  Parameter Input: x1 (kondisi awal 1), x2 (kondisi awal 2),
                   p (parameter PWLCM), beta (parameter Sine),
                   iter_awal (iterasi pemanasan), total (jumlah byte)

  Proses per-byte keystream:
    1. Fase Pemanasan: Iterasi iter_awal kali tanpa menghasilkan output
       (membuang transient awal agar sistem masuk ke rezim chaotik stabil)
    2. Sub-Sampling (N_SKIP iterasi dibuang per byte output):
       - Versi standar (_generate_keystream_core): N_SKIP = 12
       - Versi video (_generate_keystream_video): N_SKIP = 4
    3. Injeksi Umpan Balik (Feedback Injection):
       x1 = (x1 + k_prev/256) mod 1.0
       x2 = (x2 + x1) mod 1.0
    4. Evolusi terakhir sebelum ekstraksi byte
    5. Cascade XOR Folding — konversi float64 → uint8:
       val = int(x * 2^32)
       k = (val>>24 & 0xFF) ^ (val>>16 & 0xFF) ^ (val>>8 & 0xFF) ^ (val & 0xFF)
       Dilakukan untuk x1 DAN x2, lalu di-XOR: k_final = k1 ^ k2

  Fungsi Publik API:
    generate_keystream(x1,x2,p,beta,iter,total) → list[int] (mode MO-SiPINE)
    generate_keystream_sine_only(...)            → list[int] (mode Sine)
    generate_keystream_pwlcm_only(...)           → list[int] (mode PWLCM)
    generate_keystream_np(...)      → numpy uint8 array (standar, N_SKIP=12)
    generate_keystream_video_np(...) → numpy uint8 array (video, N_SKIP=4)

  Warmup: Saat modul pertama kali diimpor, fungsi _warmup() dipanggil
  otomatis untuk mengkompilasi semua varian Numba JIT sehingga tidak ada
  jeda saat pengguna pertama kali menekan tombol enkripsi.

3. ai_optimizer.py — PENCARIAN KUNCI OPTIMAL
================================================================================

Modul ini menggunakan Heuristic Search dengan Narrowing Refinement untuk
mencari parameter kunci chaos (x1, x2, p, beta) yang menghasilkan Shannon
Entropy keystream tertinggi (mendekati 8.0 bit = keacakan sempurna).

Versi saat ini: AI Optimizer v2.

3.1 ALUR KERJA (v2 — Heuristic Search + Narrowing Refinement):
  1. Generate kandidat acak: x1,x2 ∈ (0,1), p ∈ (0,0.5), beta ∈ (0.7,1.0)
  2. Filter Lyapunov: Hitung eksponen Lyapunov estimasi (n=300 iterasi, lebih
     akurat dari sebelumnya). Hanya kandidat dengan Lyapunov > 0 (chaotik)
     yang dievaluasi lebih lanjut.
  3. Bangkitkan keystream sampel (50.000 byte) dan hitung Shannon Entropy.
     (sample lebih besar dari sebelumnya agar estimasi entropy lebih stabil)
  4. Jika entropy meningkat → simpan kandidat terbaik, beralih ke Fase Lokal
     (Narrowing Refinement: perturbasi Gaussian dengan std=0.05 di sekitar
     kandidat terbaik untuk konvergensi lebih cepat).
  5. Jika _REFINE_INTERVAL=10 iterasi berturut-turut tidak ada peningkatan,
     kembali ke Fase Global (random search penuh).
  6. Early exit jika entropy >= 7.999.
  7. Timeout default: 10 detik.

3.2 KONSTANTA KONFIGURASI:
  _REFINE_STD      = 0.05   — Standar deviasi Gaussian saat fase lokal
  _REFINE_INTERVAL = 10     — Jumlah iterasi gagal sebelum restart global

3.3 FUNGSI UTAMA:
  predict_best_keys(mode, timeout, sample_size) → (best_keys: dict, entropy: float)
  calculate_entropy(data) → float (Shannon Entropy menggunakan np.bincount)
  _lyapunov_estimate(x0, p, beta, n=300) → float (estimasi eksponen Lyapunov)
  _perturb_near(best_keys, mode) → tuple (Gaussian perturbation narrowing)

4. video_core.py — ENKRIPSI/DEKRIPSI VIDEO+AUDIO
================================================================================

Backend untuk memproses video dan audio secara terpadu dalam satu file
VEN2 (.bin) dengan header kustom VEN2 (memiliki IV).

4.1 ENKRIPSI (encrypt_video_audio_to_VEN2):
  Input: file video (.mp4/.avi/.mov/.mkv)
  Output: file biner terpadu (.bin)

  Proses:
    1. Buka video dengan OpenCV, baca metadata (width, height, fps, total_frames)
    2. Ekstrak audio ke WAV menggunakan ffmpeg (jika ada stream audio)
    3. Bangkitkan angka acak 4-byte (IV) untuk mitigasi Two-Time Pad.
    4. Tulis header VEN2 36 byte (termasuk IV) ke file output
    5. Enkripsi video frame-per-frame:
       - Kunci video disuntik dengan modifikasi IV.
       - Baca frame → flatten ke 1D array
       - Bangkitkan keystream sepanjang width*height*3 byte
       - XOR: cipher = plaintext ^ keystream
       - Tulis cipher ke file
       - State Chaining antar-frame:
         x1 = (x1 + 0.0019) mod 1.0
         x2 = (x2 + 0.0073) mod 1.0
         iter_bakar: 1000 (frame pertama) → 10 (frame selanjutnya)
    6. Enkripsi audio (jika ada):
       - Hitung parameter kunci baru menggunakan kebalikan IV (pemisahan domain).
       - Bangkitkan keystream audio secara independen dari jumlah frame video.
       - XOR seluruh byte audio dengan keystream audio.
       - Tulis cipher audio ke akhir file

4.2 DEKRIPSI (decrypt_VEN2_to_video_audio):
  Input: file .bin terenkripsi + kunci parameter yang sama
  Output: file video .mp4 (+ audio jika ada)

  Proses identik dengan enkripsi (XOR simetris):
    1. Baca header dari file biner. Harus berupa header VEN2 (36 byte).
    2. Ekstrak IV dan siapkan parameter kunci terpisah untuk video dan audio.
    3. Dekripsi video frame-per-frame (XOR dengan keystream video yang sama)
    4. Tulis frame langsung ke proses FFmpeg via stdin (LOSSLESS)
    5. Dekripsi audio (XOR dengan keystream audio) → simpan WAV → mux via ffmpeg

  Codec Output (LOSSLESS):
    Sistem menggunakan FFmpeg dengan encoder libx264rgb (jika tersedia) atau
    libx264 + yuv444p sebagai fallback. Kedua opsi ini bersifat lossless,
    sehingga MSE antara video asli dan terdekripsi = 0.0 dan PSNR = infinity.
    Ini membuktikan secara matematis bahwa algoritma kriptografi MO-SiPINE
    bersifat bit-perfect (reversible sempurna).

4.3 FUNGSI PENDUKUNG:
  export_bin_to_noise_video() — Ekspor .bin langsung ke .mp4 noise (tanpa dekripsi,
                                menggunakan FFmpeg lossless libx264rgb)
  decrypt_VEN2_to_video()    — Fungsi fallback lama (format non-VEN2, FFmpeg lossless)
  extract_sample_frames()     — Ekstrak 3 frame sampel (10%, 50%, 90%)
  extract_single_frame_from_bin() — Ekstrak 1 frame dari .bin berdasarkan rasio posisi

5. audio_core.py — ENKRIPSI/DEKRIPSI AUDIO STANDALONE
================================================================================

Backend untuk enkripsi/dekripsi file audio WAV secara mandiri.

5.1 FORMAT FILE:
  Header AUENC (24 byte):
    [Magic "AUENC" 5B][SampleRate 4B][Channels 2B][NumSamples 4B]
    [BitDepth 2B][StereoFlag 2B][Padding 5B]

5.2 PROSES ENKRIPSI (encrypt_audio):
  1. Baca WAV → normalize ke int16
  2. Flatten ke byte array
  3. Bangkitkan keystream (iter_bakar=1000)
  4. XOR: cipher = raw_bytes ^ keystream
  5. Tulis header + cipher ke .bin

5.3 PROSES DEKRIPSI (decrypt_audio):
  Identik dengan enkripsi (XOR simetris). Baca .bin → XOR → simpan WAV.

5.4 FUNGSI PENDUKUNG:
  extract_audio_from_video()    — Ekstrak audio dari video via ffmpeg (PCM 16-bit, 44100 Hz)
  audio_to_noise_wav()          — Konversi cipher .bin audio ke WAV noise
  mux_video_audio()             — Gabung video+audio via ffmpeg (codec AAC)
  mux_video_audio_lossless()    — Gabung video+audio via ffmpeg (codec FLAC, lossless)
  demux_video_audio()           — Pisahkan video dan audio dari file terenkripsi

7.3 MODUL UJI HISTOGRAM & KORELASI (uji_histogram_korelasi_video):
  Window Pilih File: 1020×560 px
  Input: 2 file (Citra 1 dan Citra 2), masing-masing diekstrak 10 key-frame
  Output Window (1600×1000 px):
    - Histogram RGB untuk kedua citra (bins=256, range 0-255)
    - Tabel korelasi spasial Pearson (Horizontal, Vertikal, Diagonal)

    - Nilai MSE dan PSNR
  Catatan: Menggunakan PIL.Image.NEAREST untuk resize ke (320×240)
  (bukan LANCZOS) agar distribusi pixel cipher tidak diinterpolasi

  Output Window (1000×870 px):

    - Setiap scatter memplot SELURUH piksel frame penuh (tanpa subsampling)
    - Parameter: s=5.0, alpha=1.0 → full block solid tanpa spot putih
    - Ciphertext ideal: scatter tersebar merata di seluruh area 256×256
    - Plaintext ideal: scatter membentuk garis diagonal (korelasi tinggi)

7.5 MODUL UJI STATISTIK (uji_statistik_video):
  Window Pilih File: 1020×560 px
  Input: 2 file (Citra 1 dan Citra 2), masing-masing diekstrak 10 key-frame
  Output Window (960×400 px): Tabel metrik (rata-rata 10 key-frame):
    - NPCR (%) — Number of Pixels Change Rate. Ideal >= 99.6%
    - UACI (%) — Unified Average Changing Intensity. Ideal ≈ 33.4%
    - Entropi (rata-rata 3 channel) — Shannon Entropy. Ideal = 8.0
    - Korelasi Temporal — Pearson antar frame t dan t+1. Ideal ≈ 0.0 (cipher)
    - Chi-Square R/G/B — Uji distribusi seragam per channel
  Catatan: NPCR dan UACI dihitung berpasangan (Citra 1 vs Citra 2),
  sedangkan Entropi dan Chi-Square dihitung per citra secara independen.

8. generator_nist.py — GENERATOR BITSTREAM UNTUK NIST SP 800-22
================================================================================

Skrip mandiri untuk membangkitkan bitstream 1 Megabit (125.000 byte)
yang dapat diuji dengan software NIST Statistical Test Suite.

Proses:
  1. Validasi Lyapunov (harus > 0)
  2. Bangkitkan keystream 125.000 byte via generate_keystream_video_np
     (N_SKIP=4, konsisten dengan enkripsi video aktual)
  3. Simpan ke .txt (string '0'/'1') dan .bin (raw bytes)
  4. Hitung statistik: rasio bit 0/1, Shannon Entropy, Autocorrelation lag-1

Parameter Default:
  x1=0.123456789, x2=0.987654321, p=0.3678, beta=0.99, iter_buang=1000

Fungsi Utama:
  validasi_entropy_awal() — Cek kondisi awal sebelum bangkitkan stream masif.
  hitung_statistik_bitstream() — Mengevaluasi rasio 0/1, entropy, autokorelasi.
  hitung_progress() — Utility untuk menampilkan persentase terminal dan sisa waktu.

12. FORMAT FILE BINER KUSTOM
================================================================================

A. FORMAT VEN2 (Video Terenkripsi) — 36 byte header (VEN2):
   Sistem yang baru menggunakan Magic "VEN2" dengan tambahan 4 byte IV untuk
   Semantic Security dan mencegah serangan Two-Time Pad.
   
   Offset  Ukuran  Tipe     Isi
   0       4       bytes    Magic "VEN2"
   4       4       uint32   Width (lebar frame)
   8       4       uint32   Height (tinggi frame)
   12      4       uint32   Total Frames
   16      4       float32  FPS
   20      4       uint32   Audio Sample Rate
   24      2       uint16   Audio Channels
   26      4       uint32   Num Audio Samples
   30      2       uint16   Has Audio (0/1)
   32      4       uint32   Initialization Vector (IV) - khusus VEN2
   [36..]                   Data video terenkripsi (frame per frame)
   [setelah video]          Data audio terenkripsi (jika has_audio=1)

B. FORMAT AUENC (Audio Terenkripsi) — 24 byte header:
   Offset  Ukuran  Tipe     Isi
   0       5       bytes    Magic "AUENC"

13. METRIK EVALUASI & RUMUS
================================================================================

A. Shannon Entropy (per channel):
   H = -Σ p(i) * log2(p(i)), untuk i = 0..255
   Ideal untuk ciphertext: 8.0 bit (distribusi seragam sempurna)

B. NPCR (Number of Pixels Change Rate):
   NPCR = (Jumlah piksel yang berbeda / Total piksel) × 100%
   Ideal: >= 99.6094% (Wu et al., 2011)

C. UACI (Unified Average Changing Intensity):
   UACI = (1/N) × Σ |C1(i) - C2(i)| / 255 × 100%
   Ideal: ≈ 33.4635% (Wu et al., 2011)

D. Chi-Square Test (Distribusi Seragam):
   χ² = Σ (O_i - E)² / E, untuk i = 0..255
   E = N/256 (ekspektasi distribusi seragam)
   Nilai kecil → distribusi seragam (ciphertext baik)

E. Korelasi Spasial Pearson:
   r = Σ(x-x̄)(y-ȳ) / √(Σ(x-x̄)² × Σ(y-ȳ)²)
   Dihitung untuk arah: Horizontal, Vertikal, Diagonal
   Plaintext: mendekati 1.0 (korelasi tinggi)
   Ciphertext: mendekati 0.0 (tidak berkorelasi)

F. Korelasi Temporal:
   Pearson correlation antara Frame(t) dan Frame(t+1)
   Plaintext: mendekati 1.0 (frame berurutan mirip)
   Ciphertext: mendekati 0.0 (frame berurutan tidak berkorelasi)

G. MSE (Mean Squared Error):
   MSE = (1/N) × Σ (P_i - C_i)²
   Ideal untuk enkripsi: nilai tinggi (berbeda jauh dari asli)

H. PSNR (Peak Signal-to-Noise Ratio):
   PSNR = 20 × log10(255 / √MSE) dB
   Ideal untuk enkripsi: nilai rendah (<10 dB)

14. CATATAN TEKNIS PENTING
================================================================================

A. INTERPOLASI RESIZE:
   Saat menganalisis frame cipher untuk uji statistik, WAJIB menggunakan
   PIL.Image.NEAREST (bukan LANCZOS). LANCZOS menginterpolasi/merata-ratakan
   nilai piksel tetangga sehingga distribusi cipher yang seharusnya seragam
   menjadi terpusat di tengah (bell-curve). Ini menyebabkan:
   - Entropy turun artifisial dari ~8.0 ke ~6.3
   - Histogram terlihat bell-curve (padahal seharusnya flat)
   - UACI terukur lebih rendah dari seharusnya

B. CODEC VIDEO OUTPUT (LOSSLESS):
   Mulai versi terbaru, dekripsi menggunakan FFmpeg dengan encoder:
     - libx264rgb (prioritas utama): Encode langsung di ruang warna BGR,
       TANPA konversi ke YUV. Tidak ada rounding error sama sekali.
     - libx264 + yuv444p (fallback otomatis): Jika libx264rgb tidak tersedia,
       sistem menggunakan libx264 dengan pixel format yuv444p (4:4:4),
       yang menjamin tidak ada chroma subsampling sehingga tetap lossless.
   Dengan kedua opsi ini:
     MSE antara video asli dan terdekripsi = 0.0
     PSNR = infinity
   Ini membuktikan secara matematis bahwa algoritma MO-SiPINE bersifat
   bit-perfect (reversible sempurna). Ukuran file output lebih besar
   dibanding codec lossy (XVID), namun ini adalah trade-off yang wajar
   dan dapat dijelaskan dalam konteks penelitian akademis.

C. STATE CHAINING ANTAR-FRAME:
   Untuk mencegah pola berulang antar-frame, parameter x1 dan x2
   digeser setiap frame: x1 += 0.0019, x2 += 0.0073 (mod 1.0).
   iter_bakar juga dikurangi dari 1000 (frame pertama) ke 10
   (frame selanjutnya) untuk efisiensi.

D. AUDIO ENKRIPSI TERPISAH (DOMAIN SEPARATION & IV):
   Mulai versi terbaru, sistem memisahkan domain keystream antara audio
   dan video menggunakan Initialization Vector (IV). Parameter kunci (x1, x2)
   di-XOR dengan operasi matematika IV yang berlawanan antara video dan audio.
   Hal ini memitigasi kerentanan Two-Time Pad sepenuhnya, sambil tetap
   memastikan dekripsi audio bersifat independen (tidak bergantung pada
   jumlah frame video).

E. PENANGANAN NaN:
   Fungsi _pearson_v dan _pearson_temporal memiliki penanganan khusus
   untuk kasus varians nol (frame statis/solid) yang menyebabkan
   scipy.stats.pearsonr mengembalikan NaN.

F. KEY-FRAME SAMPLING:
   Modul uji statistik dan histogram mengekstrak 10 pasang key-frame
   (frame t dan t+1) secara merata di sepanjang durasi video, bukan
   hanya 3 frame statis. Ini memberikan evaluasi yang lebih representatif
   dan sesuai standar literatur enkripsi video digital.

G. PERFORMA:
   Setelah proses enkripsi/dekripsi selesai, sistem menampilkan:
   - FPS (Frames Per Second): total_frames / durasi
   - Throughput (MB/s): ukuran_file / durasi

H. PEMILIHAN MODE PARAMETER ENKRIPSI (AI vs MANUAL):
   Pada window enkripsi, pengguna dapat memilih dua mode input parameter
   melalui radio button:
     1. Otomatis (AI Predictor): Sistem menjalankan Heuristic Search v2 +
        Narrowing Refinement + filter Lyapunov (n=300) selama 10 detik untuk
        mencari parameter chaos yang menghasilkan entropy keystream tertinggi
        (mendekati 8.0 bit). Sampel keystream yang dievaluasi: 50.000 byte.
     2. Manual: Pengguna memasukkan nilai x1, x2, p, beta secara manual.
        Saat proses enkripsi dimulai, sistem secara otomatis menghitung
        skor entropi dari parameter yang dimasukkan menggunakan sampel
        keystream 10.000 byte, kemudian menampilkannya di kolom "Entropi Info".
        Hal ini memberikan umpan balik kualitas kriptografis tanpa memaksa
        pengguna menunggu proses AI yang lebih lama.
   Pada mode Manual, field input x1/x2/p/beta berstatus 'normal' (dapat diketik),
   sedangkan field Entropy tetap 'readonly' (diisi otomatis oleh sistem).
   Pada mode AI, semua field berstatus 'readonly' (diisi hasil prediksi AI).

I. LABEL MODE DI GUI vs KODE INTERNAL:
   Tabel pemetaan label tampilan GUI ke nilai mode internal (string):
     GUI Label         | mode string internal
     "SIP Map"         | "mosipine"  (mode default MO-SiPINE)
     "Sine Only"       | "sine"
     "PWLCM Only"      | "pwlcm"
     "MO-SiPINE"       | "mosipine"  (digunakan di window Dekripsi)
   Label "SIP Map" adalah nama singkat yang digunakan di window Enkripsi
   dan Enkripsi Audio Standalone. Window Dekripsi menggunakan label
   "MO-SiPINE" untuk konsistensi dengan terminologi akademis.
